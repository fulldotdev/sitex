## Overview

Every page's frontmatter is available through `sitex:pages`, so indexes, cards, sidebars, and navigation can use page-owned values. Layouts and components query it — pages stay plain content documents.

## getPages

Use `getPages` to read all pages, or pass a slash-prefixed path prefix.

```tsx
import { getPages } from "sitex:pages"

export default async function BlogIndex() {
  const posts = await getPages("/blog")

  return (
    <main>
      <h1>Blog</h1>
      {posts.map((post) => (
        <article key={post.path}>
          <a href={post.path}>{post.title}</a>
          <p>{post.description}</p>
        </article>
      ))}
    </main>
  )
}
```

`page.path` follows the configured trailing-slash style.

## getPage

Use `getPage` when you know the route path.

```tsx
import { getPage } from "sitex:pages"

export default async function FeaturedPost() {
  const post = await getPage("/blog/building-content-sites")

  if (!post) return null

  return <a href={post.path}>{post.title}</a>
}
```

The return value is `Page | undefined`.

## The Page Type

Because every page follows the same frontmatter schema, `Page` is a stable type — no code generation involved.

```ts
type Page = {
  path: string
  layout: string
  title: string
  description: string
  type?: "webpage" | "article" | "faq"
  image?: string
  canonical?: string
  noindex?: boolean
  publishedAt?: string
  updatedAt?: string
  author?: string
  questions?: FaqQuestion[]
  headings: MarkdownHeading[]
  [key: string]: JsonValue | undefined
}
```

Extra frontmatter fields are included with the `JsonValue` index signature. Narrow them where you use them, or wrap `getPages` in an app-owned helper that asserts your shape.

## Async Components

`getPages` and `getPage` are async, so use them in async components. Every page renders at build time, so async components are fine anywhere in a layout tree.
