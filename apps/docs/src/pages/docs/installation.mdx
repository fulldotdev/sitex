## AI Agent Install

Give your agent this instruction from inside the project you want to set up:

```text
Read https://sitex.full.dev/docs/installation and install Sitex in this project.

Follow the manual install steps on that page: install the package, add the Vite plugin
with the site config, create global CSS, extend the Sitex TypeScript config, and create
the first layout and page.
```

## Manual Install

Install Sitex, React, Vite+, and the Vite packages your app needs. The example below includes Tailwind; omit `tailwindcss`, `@tailwindcss/vite`, and the Tailwind plugin if your app uses another CSS setup.

```bash
pnpm add @fulldotdev/sitex react react-dom vite vite-plus @vitejs/plugin-react tailwindcss
pnpm add -D @tailwindcss/vite
pnpm add -D typescript @types/react @types/react-dom
```

## Vite Config

Add the Sitex plugin to `vite.config.ts`. The `site` config only needs `url`; Sitex uses it for sitemap and robots output. Sitex sets Vite's custom app mode. Add the Tailwind plugin only when the app uses Tailwind.

```ts
import { defineConfig } from "vite-plus"
import { sitex } from "@fulldotdev/sitex/plugin"
import tailwindcss from "@tailwindcss/vite"
import react from "@vitejs/plugin-react"

export default defineConfig({
  plugins: [
    react(),
    sitex({
      site: {
        url: "https://example.com",
      },
    }),
    tailwindcss(),
  ],
})
```

Sitex writes slashless URLs by default. Pass `trailingSlash: true` if you want directory index output and trailing-slash paths. Read [SEO and head](/docs/seo) for the recommended app-owned metadata component.

## Global CSS

Create `src/index.css` and import it from your base layout.

```css
@import "tailwindcss";
@import "@fontsource-variable/geist";

:root {
  --font-sans: "Geist Variable", sans-serif;
}
```

```tsx
// src/layouts/base.tsx
import "@/index.css"
```

Add app-wide CSS, font imports, theme tokens, and Tailwind layers here. If your app uses Tailwind, install `tailwindcss` and `@tailwindcss/vite`, add the Tailwind Vite plugin, and import Tailwind from this file.

## Global Content

Create `src/globals/index.yaml` for shared site chrome content. The root locale name comes from `site.locale` (default `"en"`) and sets `<html lang>`; files like `src/globals/nl.yaml` define locale variants served under `/nl` routes. Layouts import `sitex:globals` for header, sidebar, footer, logo, and shared labels.

```yaml
name: My Site
logo:
  label: My Site
  href: /
header:
  navigation:
    - label: Home
      href: /
    - label: Docs
      href: /docs
```

```tsx
import globals from "sitex:globals"

export default function DefaultLayout({ children }: LayoutProps) {
  return (
    <>
      <header>
        <a href={globals.logo.href}>{globals.logo.label}</a>
      </header>
      <main>{children}</main>
    </>
  )
}
```

The Vite `site` config is for document metadata and derived SEO output. `src/globals/index.yaml` is for visible, reusable site content.

## TypeScript Config

Extend the Sitex TypeScript config so JSX client directives are accepted and generated page typechecks are included.

```json
{
  "extends": "@fulldotdev/sitex/tsconfig",
  "include": [
    "src/**/*",
    ".sitex/**/*.d.ts",
    ".sitex/typecheck/**/*.tsx",
    "vite.config.ts"
  ],
  "exclude": ["dist"],
  "compilerOptions": {
    "paths": {
      "@/*": ["./src/*"]
    }
  }
}
```

Add the generated `.sitex` directory to `.gitignore` — Sitex rewrites it on every dev run and build.

## First Layout

Create `src/layouts/default.tsx`. Layouts return page content.

```tsx
import type { LayoutProps } from "@fulldotdev/sitex"

export default function DefaultLayout({ title, children }: LayoutProps) {
  return (
    <main>
      <h1>{title}</h1>
      {children}
    </main>
  )
}
```

## First Page

Create `src/pages/index.mdx`.

```mdx
---
layout: "default"
title: "My Site"
description: "My first Sitex page."
---

Hello from Sitex.
```

`src/pages/index.mdx` becomes `/`. Read [Pages](/docs/pages) for the frontmatter schema.

## Favicon

Sitex generates `/favicon.svg` when you do not provide `public/favicon.svg`.

## Build Behavior

Sitex runs through Vite+.

```bash
vp dev
vp build
vp preview
```

`vp dev` starts the development server. `vp build` processes the framework-owned Vite HTML entry, writes one HTML file per page with the app-rendered head and body, plus `sitemap.xml` and `robots.txt`, and adds the island client script only when a page needs client rendering. `vp preview` serves the built output locally.
