## Required Folders

Sitex requires two folders. Global CSS and global site chrome have standard app locations.

1. `src/index.css` is the recommended global stylesheet. Import it from `src/layouts/base.tsx`.
2. `src/globals/index.yaml` holds global content used by layouts, such as logo, header navigation, sidebar navigation, footer links, and labels. Additional files like `nl.yaml` define locale variants served under `/nl` routes.
3. `src/pages` holds MDX pages. Every `.mdx` file becomes a route. TSX files are not allowed here.
4. `src/layouts` holds TSX layouts. `layout: "doc"` uses `src/layouts/doc.tsx`, `layout: "blog/post"` uses `src/layouts/blog/post.tsx`.

## Assets

Put app-owned images in `src/assets`. Use `public/` only for static files that must keep an exact public URL. Sitex generates `/favicon.svg` unless the app provides `public/favicon.svg`.

## Recommended Shape

Keep pages thin: frontmatter facts plus Markdown content. Layouts own structure and chrome. Put shared chrome content in `src/globals/index.yaml`, then import `sitex:globals` from layouts and blocks.

Use `src/components` for reusable implementation pieces. This folder is a convention, not a Sitex requirement.

## Blocks

Place blocks in `src/components/blocks`. Blocks are reusable page sections, such as hero sections, feature grids, article frames, and sidebars.

## UI

Place UI components in `src/components/ui`. UI components are low-level design system pieces, such as buttons, typography, inputs, and sidebar primitives.
