## Overview

Sitex renders every page to HTML at build time. Vite processes the framework-owned HTML entry, then Sitex injects the app-rendered head and body into that transformed template.

Read [SEO and head](/docs/seo) for the recommended app-owned metadata component.

## Static Output

`vp build` writes one HTML file per page, plus `sitemap.xml` and `robots.txt`. There is no server at request time — deploy the `dist` folder to any static host.

## CSS Assets

Create `src/index.css` and import it from your base layout. Tailwind, CSS modules, plain CSS, or another CSS tool can live in the app.

```css
body {
  margin: 0;
}
```

Route-specific CSS can still be imported from layouts or components. Sitex follows each page's module graph and includes the CSS that page needs.

## Island Script

Pages ship zero JavaScript by default. When a page contains at least one island, Sitex adds the island client script to that page only. Read [Component rendering](/docs/island-rendering).

## Development Server

`vp dev` renders pages on the fly with the same document pipeline, so the head you see in development matches the build output.
