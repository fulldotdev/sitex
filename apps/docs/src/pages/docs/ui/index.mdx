Sitex UI is a set of components built on the [shadcn/ui](https://ui.shadcn.com) conventions: Tailwind styling, `cn` from `@/lib/utils`, and your theme tokens. It exists because content sites keep needing the same pieces — sections, heroes-adjacent primitives, prose typography, code blocks, a table of contents, a price display — and those pieces don't belong in the framework. Sitex stays a rendering pipeline; the design layer lives in your app.

That is also why it ships as a registry instead of an npm package. You install a component and its source lands in `src/components/ui`, where you own it: restyle it, extend it, or delete half of it. There is no version to track and no API boundary to fight — the same model as shadcn/ui, and the components compose with regular shadcn/ui components in the same app.

Start with [Installation](/docs/ui/installation), then browse the [component overview](/docs/ui/components).
