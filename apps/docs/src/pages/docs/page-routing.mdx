## Overview

Sitex uses file-based routing. TSX and MDX files in `src/pages` become routes. This is the required part of the folder structure explained in [Folder structure](/docs/folder-structure).

Routes are static unless they export `render = "server"`. See [Rendering and assets](/docs/rendering-and-assets) for how static and server routes are rendered.

## Route files

A TSX route exports the page component for one URL. An MDX route uses frontmatter to choose a layout and renders its body as that layout's children.

1. `src/pages/about.tsx` becomes `/about`.
2. `src/pages/docs/page-routing.mdx` becomes `/docs/page-routing`.

## Index routes

An `index.tsx` or `index.mdx` file maps to its parent folder. This keeps root pages and section landing pages explicit.

1. `src/pages/index.tsx` becomes `/`.
2. `src/pages/docs/index.mdx` becomes `/docs`.

## Dynamic routes

A route segment wrapped in brackets becomes a route parameter. Static dynamic routes must export `paths`, so Sitex knows which HTML files to build.

1. `src/pages/blog/[slug].tsx` can become `/blog/hello-world`.
2. Each entry in `paths` provides the params for one generated page.

## Server routes

Export `render = "server"` for request-time routes. Server routes can read the incoming `request` from page context and are not written as static HTML files.
