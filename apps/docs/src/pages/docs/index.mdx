## Overview

Sitex is an opinionated React framework for content sites. Pages are MDX documents with typed frontmatter. Structure, chrome, and design live in TSX layouts and components. Every page renders to static HTML at build time.

The framework handles the boring parts that should always be the same: routing, Vite HTML transforms, asset loading, island scripts, sitemap, and robots.txt. Your layouts render document tags, metadata, and CSS imports with normal TSX. Read [SEO and head](/docs/seo).

Imported components can opt into client rendering with directives like `client:load`, `client:visible`, and `client:idle`. Read [Component rendering](/docs/island-rendering).

## The Model

1. `src/pages/**/*.mdx` — routable content documents with typed frontmatter.
2. `src/layouts/**/*.tsx` — page structure and chrome, selected by the `layout` field.
3. `src/components/**/*.tsx` — blocks, UI primitives, and islands.
4. Site config in `vite.config.ts` — stable site facts for output, sitemap, robots, and app helpers.

This split is strict on purpose. Because every page is a content document flowing through the same pipeline, metadata and structured data are always complete — and pages are simple enough that an AI agent or a teammate can add one without touching React.

## Use Cases

Sitex is intended for content-based websites: landing pages, marketing sites, documentation, and blogs. It is especially useful when the content site belongs to a larger product that also has an app-like dashboard built with something like Next.js or TanStack, because both can share the same React components and design system.

## Sitex Versus Astro

We love Astro. Fulldev sponsors Astro monthly, and Sitex is mostly inspired by Astro's static-first model and island architecture.

Sitex differs in two ways. First, everything around content is React: layouts, blocks, UI, metadata, and islands are plain TSX with client directives, so there is no second component syntax. Second, Sitex keeps routes as MDX documents while leaving app-specific head composition in your layouts.
